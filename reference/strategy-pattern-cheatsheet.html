Strategy Pattern — Cheatsheet

Tóm tắt 1 trang. In ra dán tường nếu cần.

Bốn thành phần

Vai tròTrong bài toánTrách nhiệm
Strategy TargetSelector (interface) Định nghĩa hành vi có thể thay thế
ConcreteStrategy NearestEnemy, LowestHP, MarkedTarget Một cách cụ thể để làm hành vi đó
Context Hero Giữ reference tới Strategy, ủy quyền công việc
Client Code khởi tạo new Hero("Yena", new LowestHP()) Chọn ConcreteStrategy nào gắn vào Context

Skeleton Java — copy và sửa

// 1. Strategy
public interface TargetSelector {
    Enemy select(Enemy attacker, List<Enemy> enemies);
}

// 2. ConcreteStrategy
public class NearestEnemy implements TargetSelector {
    @Override public Enemy select(Enemy a, List<Enemy> es) {
        // ...
    }
}

// 3. Context
public class Hero {
    private final TargetSelector selector;
    public Hero(TargetSelector s) { this.selector = s; }
    public void autoAttack(List<Enemy> enemies) {
        Enemy t = selector.select(this, enemies);
        if (t != null) System.out.println("hit " + t);
    }
}

// 4. Client
Hero yena = new Hero(new LowestHP());

Quy tắc vàng

  • Context không được biết ConcreteStrategy cụ thể — chỉ giữ reference tới interface.
  • Mỗi ConcreteStrategy là một class riêng — không switch/if bên trong Strategy.
  • Strategy có thể trả null — caller quyết định xử lý.
  • Strategy có thể được thay đổi runtime qua setter, không chỉ constructor.
  • Đừng nhầm Strategy với State — Strategy do Client chọn; State do Context tự chuyển.

So sánh với các lựa chọn khác

CáchKhi nào dùngVấn đề
if/else trong Hero 1-2 trường hợp, không mở rộng Thêm case = sửa Hero. Vi phạm Open/Closed.
switch trên enum Enum cố định, ít case Thêm enum = sửa switch. Vẫn vi phạm O/C.
Inheritance (LowestHPHero extends Hero) Phân cấp tĩnh, ít biến thể Bùng nổ tổ hợp khi có N chiều biến thể.
Strategy pattern Nhiều thuật toán cho cùng hành vi, mở rộng thường xuyên Thêm class. Nhiều file nhỏ.
